## 1. Home 탭

### 1-1. Summary Cards 개선

**현재 문제**: 3개의 카드가 동일한 크기로 숫자만 표시하고 있다. 아이콘도 흐릿한 회색이라 시각적 임팩트가 약하다. "4,451"이라는 숫자가 얼마나 큰 건지, 늘고 있는 건지에 대한 맥락이 전혀 없다.

**개선 방안**:
- 각 카드에 **전월 대비 증감률** 또는 **최근 이벤트 기준 증감**을 추가한다. 예: "4,451 fighters (+23 this month)" — 데이터가 살아있다는 느낌을 준다
- 아이콘에 각 카드별 **고유 accent 색상**을 부여한다. 현재 아이콘이 모두 동일한 회색이라 구분이 안 됨
- 4번째 카드를 추가하는 것을 고려한다. 예: "Avg Finish Rate" 또는 "Latest Event" — 3열보다 4열이 시각적으로 안정적이고, 나머지 탭들의 4개 카드 그리드와도 일관된다

### 1-2. Recent Events / Upcoming Events 레이아웃

**현재 문제**: Recent Events가 왼쪽에서 세로로 길게 5개 나열되어 있고, Upcoming Events는 "No upcoming events" 한 줄로 오른쪽 절반 전체가 비어있다. 이 빈 공간이 페이지 전체를 허전하게 만든다.

**개선 방안**:
- **Upcoming Events가 비어있을 때 대체 콘텐츠**를 표시한다. 예: "가장 최근 이벤트 하이라이트", "이번 달 주요 통계", 또는 다음 이벤트 일정이 확정되면 알림을 받을 수 있는 안내 등
- Recent Events 카드에 **승리 방법 뱃지**를 추가한다. 현재 "14 fights"만 있는데, "7 KO/TKO · 4 DEC · 3 SUB" 같은 간단한 breakdown을 넣으면 정보 밀도가 높아진다
- 메인 이벤트 선수 이름이 전부 소문자("justin gaethje vs paddy pimblett")인데, **첫 글자 대문자(Title Case)로 변환**해야 한다. 이건 전체 대시보드에 걸친 문제이다 (아래 3-2에서 상세 설명)

### 1-3. Event Locations 지도

**현재 상태**: 다크 테마 지도에 보라색 점으로 이벤트 위치를 표시하고 있다. 시각적으로 깔끔하다.

**개선 방안**:
- 점 크기가 이벤트 수를 반영하고 있는 것 같지만, **범례(legend)가 없다**. "점 크기 = 이벤트 수"라는 것을 알려주는 범례를 추가한다
- 줌인/줌아웃 컨트롤(+/- 버튼)의 스타일이 지도 라이브러리 기본값(흰색 사각형)으로 되어 있어 다크 테마와 어울리지 않는다. 커스텀 스타일링이 필요하다
- 점에 hover 시 "Las Vegas, NV — 187 events" 같은 툴팁이 있으면 좋다

### 1-4. Nationality Distribution

**현재 상태**: Top 15 + Others로 수평 바 차트를 표시하고 있다.

**개선 방안**:
- "Others" 바가 가장 길어서 다른 나라들의 바가 상대적으로 너무 짧게 보인다. **Others를 별도 처리**(다른 색상이나 분리선)하면 실제 국가별 비교가 더 쉬워진다
- 국기 이모지를 선수 이름 옆에 추가하면 시각적 흥미가 올라간다. 예: "🇺🇸 United States", "🇧🇷 Brazil"
- USA가 압도적으로 많을 텐데 차트에서 잘려있다. **1위 국가의 바에 수치 라벨**을 직접 표시하면 좋다

## 2. 색상 시스템 및 시각적 통일성

### 2-1. 탭별 색상 불일치

**현재 문제**: 각 탭마다 차트 색상이 제각각이다.
- Overview: 보라색(Weight Class Activity) + 오렌지(Fight Duration) + 멀티컬러(Finish Rate Trend)
- Striking: 빨간색/코랄(KO/TKO, Knockdown) + 오렌지(Sig. Strikes by Weight) + 에메랄드(Strike Exchange)
- Grappling: 시안(Control Time, TD) + 에메랄드(Ground Strikes, TD Defense) + 보라색(Submission)

색상 자체는 나쁘지 않지만, **의미 체계(semantic color system)가 없다**. 동일한 데이터 유형에 다른 색이 쓰이거나, 다른 유형에 같은 색이 쓰인다.

**개선 방안**:

카테고리별로 색상을 할당하는 **시맨틱 컬러 시스템**을 정의한다:

| 카테고리 | 권장 색상 | 현재 사용 |
|---------|----------|----------|
| KO/TKO 관련 | 빨간계열 통일 | 이미 빨간/코랄 사용 중 ✓ |
| 서브미션 관련 | 보라계열 통일 | 부분적으로 사용 중 |
| 판정 관련 | 시안/블루 계열 | 일관성 없음 |
| 테이크다운 관련 | 에메랄드 계열 | 부분적으로 사용 중 |
| 타격 관련 | 오렌지/앰버 계열 | 부분적으로 사용 중 |
| 체급 비교 | 보라 그래디언트 | 사용 중 ✓ |

이렇게 하면 사용자가 색상만 보고도 "아, 이건 타격 관련 데이터구나"라고 직관적으로 인식할 수 있다.

### 2-2. Finish Rate Trend 차트 색상

**현재 문제**: 이 차트에서 Decision=보라, KO/TKO=빨강, Submission=시안을 사용하고 있다. 그런데 초기 연도(1994~2005)에서 선들이 많이 겹치고, 다크 배경에서 보라색 선이 잘 안 보인다.

**개선 방안**:
- 보라색 선을 더 밝은 톤으로 변경하거나, 선 두께를 2px → 2.5px로 키운다
- 이 차트는 영역(area)을 채우고 있는데, 영역이 겹치는 부분의 가독성이 떨어진다. **투명도를 더 낮추거나**(0.1 정도) 영역 대신 순수 라인 차트로 변경하는 것을 고려한다
- 선에 **주요 전환점(turning point)에 dot을 찍어주면** 데이터 포인트를 더 쉽게 읽을 수 있다

## 3. 차트 가독성

### 3-1. X축 레이블 겹침/잘림

**현재 문제**: 여러 차트에서 X축 레이블이 겹치거나 잘려서 읽기 어렵다.
- **Sig. Strikes by Weight Class** (Image 4 우측 상단): 체급 이름이 대각선으로 표시되어 있지만, "women's flyweight", "catch weight" 등이 서로 겹쳐서 거의 읽을 수 없다
- **Control Time** (Image 5 좌측 상단): "W.Bantam", "W.Fly" 등이 기울어져 겹친다
- **Weight Class Activity** (Image 3 좌측 상단): 비교적 낫지만 여전히 빽빽하다

**개선 방안**:
- 체급 이름을 **약어로 통일**한다. 예: "LW" (Lightweight), "WW" (Welterweight), "MW" (Middleweight) — 차트 아래에 약어 범례를 한 번만 표시
- 또는 **수평 바 차트로 전환**한다. 체급 이름이 Y축에 오면 텍스트가 겹치지 않는다. 실제로 이미 많은 리더보드 차트가 수평 바를 사용 중이므로 일관성도 좋아진다
- X축 레이블에 `interval` 속성을 활용하여 간격을 두거나, 반응형으로 화면 크기에 따라 표시 개수를 조절한다

### 3-2. 선수 이름 대소문자

**현재 문제**: 모든 선수 이름이 **전부 소문자**로 표시되어 있다. "derrick lewis", "anderson silva", "khabib nurmagomedov" 등. 이는 비전문적으로 보이고 가독성을 떨어뜨린다.

**개선 방안**:
- **Title Case로 변환**: "Derrick Lewis", "Anderson Silva", "Khabib Nurmagomedov"
- 프론트엔드에서 CSS `text-transform: capitalize` 또는 유틸 함수로 처리 가능
- 이벤트 메인 이벤트 텍스트("justin gaethje vs paddy pimblett")도 동일하게 적용 필요

### 3-3. 빈 여백과 차트 크기 밸런스

**현재 문제**: 일부 차트가 카드 안에서 여백이 과하게 남는다.
- **Stance Winrate** (Image 4 우측 하단): 3×3 매트릭스 테이블인데 카드 절반 이상이 비어있다
- **Submission Efficiency** (Image 5 우측 상단): 산점도에 점이 몇 개 안 되어 카드가 헐렁해 보인다

**개선 방안**:
- Stance Winrate 카드 크기를 줄이거나, 테이블 아래에 **인사이트 텍스트**를 추가한다. 예: "사우스포가 오소독스 상대로 53.4% 승률을 기록하고 있습니다"
- 산점도 차트는 점 개수가 적을 때 카드가 빈 느낌이 나므로, **min_fights 기준을 낮추거나** 데이터 포인트를 더 많이 표시하는 것을 고려한다

## 4. Home 탭 — Category Leaders / Rankings 개선

### 4-1. Category Leaders 카드

기획 문서에 "분야별 1등 선수 카드 (8개 분야)" 이 있다. 스크린샷에서는 안 보이지만, 이 카드들은 Home 탭의 핵심 콘텐츠가 될 수 있다.

**디자인 제안**:
- 각 분야 카드에 **아이콘 + 분야명 + 선수명 + 수치**를 넣되, 카드마다 해당 분야의 시맨틱 컬러를 은은하게 적용한다
- 8개를 4×2 그리드로 배치하면 Summary Cards 바로 아래에서 "한눈에 보는 챔피언"이라는 강한 인상을 줄 수 있다
- 선수 이름 클릭 시 해당 선수의 상세 통계로 이동할 수 있으면 좋다 (미래 기능)

### 4-2. Division Rankings

기획 문서에 "체급별 챔피언 & 랭킹"이 있다. 이것도 Home 탭에 있으면 격투기 팬들이 가장 자주 확인하는 핵심 정보이다.

**디자인 제안**:
- 체급 드롭다운 + 챔피언 카드(강조) + 1~15위 리스트
- 챔피언 카드는 골드 accent + 벨트 아이콘으로 시각적 차별화
- 랭커 리스트는 순위 숫자에 subtle한 배경을 넣어 위계를 표현

## 5. 인터랙션 및 사용자 경험

### 5-1. 차트 간 연결성 (Cross-filtering)

**현재 문제**: 각 차트가 완전히 독립적으로 동작한다. Weight Class Activity에서 "Lightweight" 바를 클릭해도 옆의 Fight Duration 차트에는 아무 변화가 없다.

**개선 방안** (장기):
- 체급 바 차트에서 특정 체급을 클릭하면, **같은 행의 다른 차트도 해당 체급으로 자동 필터링**되는 cross-filtering을 추가한다
- 즉시 구현이 어렵다면, 체급 바를 클릭했을 때 **해당 체급의 WeightClassFilter가 자동 선택**되는 것만으로도 연결성이 생긴다

### 5-2. 더보기 / 접기 패턴

**기획 문서에 있는 내용**: "TOP N — 프론트엔드에서 기본 5건만 표시, '더보기' 클릭 시 나머지 5건 노출"

스크린샷을 보면 현재 10명이 한번에 다 보이는 것 같다. 기획대로 **5명 → 더보기 → 10명** 패턴을 적용하면:
- 초기 로딩 시 카드가 더 컴팩트해져 한 화면에 더 많은 차트가 보인다
- "더보기" 버튼에 Framer Motion으로 확장 애니메이션을 넣으면 자연스럽다

### 5-3. 선수 이름 인터랙션

**현재 문제**: 리더보드의 선수 이름이 단순 텍스트이다.

**개선 방안**:
- 선수 이름에 **hover 시 underline 또는 색상 변경**을 추가하여 클릭 가능한 것처럼 표시한다
- 지금 당장 상세 페이지가 없더라도, 클릭 시 **간단한 팝오버**(전적, 체급, 스탠스 정도)를 보여주면 탐색성이 크게 올라간다
- AI Chat 기능이 있으므로, 선수 클릭 시 "이 선수에 대해 AI에게 물어보기" 같은 연결도 가능하다

### 5-4. 툴팁 개선

**현재 상태**: Recharts 기본 툴팁에 다크 테마 스타일링만 적용한 상태이다.

**개선 방안**:
- **커스텀 툴팁 컴포넌트**를 만들어 더 풍부한 정보를 표시한다
- 예를 들어 KO/TKO Leaders에서 바에 hover 하면: "Derrick Lewis — KO/TKO: 16회 (전체 승리 중 64%) — 체급: Heavyweight"
- 바 차트 툴팁에 **미니 스파크라인**(최근 5경기 결과 등)을 넣는 것도 고급 패턴이다

## 6. 레이아웃 및 그리드

### 6-1. Bento Grid 크기 변형 활용

**현재 문제**: 대부분의 차트가 **1:1 또는 1:2 비율**로만 배치되어 있다. 모든 카드가 같은 높이라 단조롭다.

**개선 방안**:
- 중요도가 높은 차트는 **2 column span** 또는 **높이를 키워서** 시각적 위계를 만든다
- 예시 배치:
  - Finish Rate Trend는 전체 너비(2 column span) — 현재도 이렇게 되어있어 좋다
  - Finish Methods(도넛) + Events Timeline(area)은 1:1인데, Events Timeline을 좀 더 넓게(3:2 비율) 해도 좋다
  - Stance Winrate(작은 테이블)는 차트와 함께 한 칸에 넣을 수도 있다

### 6-2. 탭 간 로딩 경험

**현재 문제**: 탭 전환 시 데이터를 새로 불러올 때의 경험이 스크린샷에서는 확인하기 어렵다.

**개선 방안**:
- ChartCard에 이미 Skeleton 로딩이 구현되어 있는데 (코드 확인), **Skeleton 개수와 크기를 실제 차트 형태에 맞추면** 더 자연스럽다
- 예: 도넛 차트 자리에는 원형 Skeleton, 바 차트 자리에는 여러 개의 직사각형 Skeleton
- 탭 전환 시 **이전 데이터를 즉시 지우지 말고**, 새 데이터가 도착할 때까지 이전 데이터를 흐릿하게(opacity 0.5) 보여주는 "optimistic UI" 패턴도 좋다

## 7. 타이포그래피 및 미세 디테일

### 7-1. 숫자 폰트

**현재 문제**: 숫자와 텍스트가 동일한 폰트를 사용하고 있어, 데이터 대시보드로서의 "숫자 중심" 느낌이 약하다.

**개선 방안**:
- Summary 카드의 큰 숫자, 차트 라벨의 수치에 **모노스페이스 또는 tabular figures 지원 폰트**를 사용한다
- 추천: `JetBrains Mono`, `IBM Plex Mono`, 또는 현재 폰트에 `font-variant-numeric: tabular-nums` 적용
- 숫자가 정렬될 때 자릿수가 맞아야 바 차트 라벨 등이 깔끔하게 보인다

### 7-2. 차트 제목 패턴 통일

**현재 상태**: 대부분 "Title + (i) + description" 패턴으로 잘 되어있다.

**개선 방안**:
- 일부 차트에는 description이 없다 (Control Time, Ground Strikes 등). **모든 차트에 description을 통일**하면 시각적 리듬이 좋아진다
- 차트 제목에 **현재 필터 상태**를 반영하면 좋다. 예: "KO/TKO Leaders" → 필터 변경 시 "KO/TKO Leaders — Lightweight"

## 8. Grappling 탭 특수 차트

### 8-1. Ground Strikes (Bubble Chart)

**현재 상태**: Attempted vs Landed 산점도인데, 점들이 우측 상단에 밀집되어 있고 나머지는 비어있다.

**개선 방안**:
- **로그 스케일** 또는 **축 범위 조정**으로 밀집된 영역을 확대한다
- georges st-pierre 같은 이상치(outlier)를 별도 라벨로 표시하고 있는 건 좋다. 나머지 점에도 **hover 시 선수 이름이 보이도록** 한다
- 점의 accuracy를 **색상 그래디언트**로 표현하면 (낮은 accuracy = 흐린 색, 높은 accuracy = 진한 색) 차원이 하나 더 생긴다

### 8-2. Submission Efficiency (Scatter)

**현재 상태**: 점이 적고, 트렌드 라인(점선)이 있다. charles oliveira가 우측 상단에 눈에 띈다.

**개선 방안**:
- "Avg 47%" 라벨이 좀 작다. **사분면 라벨**을 추가하면 좋다: 우측 상단 = "High Attempts, High Finishes" 영역임을 표시
- 점에 hover 시 **"시도 대비 성공 비율"**을 표시하면 더 유의미하다

### 8-3. TD Attempts Leaders (Lollipop)

**현재 상태**: 시안색 바 + 점으로 된 롤리팝 차트. "Avg 2.8" 기준선이 있다.

**개선 방안**:
- 기준선(Avg) 왼쪽과 오른쪽에 **배경 색상을 다르게** 하면 (왼쪽 = 평균 이하, 오른쪽 = 평균 이상) 직관적으로 파악된다
- 기준선 라벨 "Avg 2.8"이 X축과 겹쳐서 잘 안 보인다. 라벨 위치를 조정한다

## 9. 반응형 및 접근성

### 9-1. 모바일 대응

스크린샷은 데스크탑 기준이지만, 모바일 대응이 되어있는지 확인이 필요하다.

**체크 포인트**:
- 2 column 그리드 → 모바일에서 1 column으로 변경
- 수평 바 차트의 선수 이름(Y축 width 100~110px)이 모바일에서 공간을 많이 차지할 수 있다
- PillTabs가 3개 탭까지는 괜찮지만, 화면이 좁으면 탭 텍스트 크기 조정 필요
- 지도는 모바일에서 터치 조작이 자연스러운지 확인

### 9-2. 접근성

**체크 포인트**:
- 차트 색상이 **색각 이상자(색맹)**에게도 구분 가능한지 확인한다. 특히 빨강/초록이 함께 쓰이는 곳(Takedown Accuracy의 정확도별 색상)
- 모든 차트에 `aria-label`이 설정되어 있는지 확인한다
- HelpCircle 아이콘(?)의 툴팁이 키보드로도 접근 가능한지 확인한다 — 코드 확인 결과 `TooltipTrigger asChild`로 되어있어 기본적으로는 동작하지만, 포커스 스타일이 보이는지 확인 필요

**개선 방안**:

**현재 상태**: 상단 네비게이션에 "Dashboard"와 "AI Chat"이 분리되어 있다.

**개선 방안**:
- 페이지 하단 또는 Summary Cards 근처에 **"Data as of Jan 24, 2026"** 같은 타임스탬프를 표시한다
- 또는 Summary Cards에 "Last updated: 2 days ago" 형태로 넣는다

## 11. 선수 상세 페이지 (Fighter Detail)

> 분석 기준: Islam Makhachev 선수 상세 페이지 스크린샷

### 11-1. 프로필 헤더

**현재 문제**:
- 선수 이름이 소문자("islam makhachev")로 표시되어 있다. 대시보드와 동일한 문제이며, 선수 상세 페이지에서는 이름이 가장 큰 텍스트이므로 **체감이 더 크다**
- "men's pound-for-pound: #1"도 소문자. "Men's Pound-for-Pound: #1"으로 변환 필요
- Champion 뱃지의 초록색이 다크 테마에서 너무 밝아 눈에 거슬린다. 현재 테마의 accent 색상(보라 또는 골드)으로 맞추거나, 골드 계열(`#f59e0b`)로 변경하면 "챔피언"이라는 의미와 더 잘 맞는다
- 선수 프로필 사진이 없다. 물론 저작권 문제일 수 있지만, **실루엣 아바타** 또는 **스탠스 아이콘**(Orthodox/Southpaw 실루엣)이라도 있으면 헤더의 빈 공간이 줄어든다

**개선 방안**:
- 이름을 Title Case로 변환: "Islam Makhachev"
- 헤더 왼쪽에 **선수 아바타 placeholder**(이니셜 또는 스탠스 실루엣)를 추가한다
- Champion 뱃지를 골드 계열로 변경하고, 벨트 이모지(🏆)와 함께 표시한다
- 프로필 정보(Russia, Southpaw, 신체 스펙)의 아이콘들이 너무 작고 흐려서 잘 안 보인다. 아이콘 크기를 키우거나 색상을 약간 밝게 조정한다

### 11-2. Record 카드

**현재 상태**: 28승(초록)-1패(빨강)-0무(회색) + Win Rate 프로그레스 바 + "16 Win Streak" 뱃지

**잘된 점**: 색상으로 승패를 직관적으로 구분하고 있다. Win Streak 뱃지도 좋은 정보이다.

**개선 방안**:
- Win Rate 퍼센트(96.6%)와 프로그레스 바 사이에 **시각적 계층**이 약하다. 퍼센트 숫자를 더 크게 하거나 볼드 처리한다
- "16 Win Streak" 뱃지가 초록색인데, 이것도 챔피언 뱃지와 비슷한 톤이라 구분이 안 된다. 뱃지 색상에 의미를 부여한다: Champion = 골드, Win Streak = 에메랄드, Ranking = 보라 등
- **전적 비교 맥락**을 추가하면 좋다. 예: "체급 평균 승률: 52.3%" 같은 기준선이 있으면 96.6%가 얼마나 뛰어난지 직관적으로 파악된다

### 11-3. Finish Breakdown 도넛 차트

**현재 상태**: KO/TKO(빨강) 3건, Submission(핑크) 8건, Decision(시안) 6건

**개선 방안**:
- 도넛 차트 중앙에 **총 승수("17 Wins")를 표시**하면 공간 활용이 좋아지고, 퍼센트의 모수가 뭔지 한눈에 보인다
- 현재 색상(빨강, 핑크, 시안)이 대시보드의 Finish Methods 도넛(보라, 시안, 에메랄드, 앰버, 빨강, 핑크)과 다르다. **대시보드와 동일한 시맨틱 컬러**를 사용해야 사용자가 페이지를 넘나들 때 혼란이 없다
- 범례가 차트 오른쪽에 세로로 배치되어 있는데, 카드 내 여백이 충분하므로 괜찮다. 다만 **범례 텍스트에 hover 시 해당 섹터가 강조**되면 인터랙션이 좋아진다

### 11-4. Striking / Grappling 통계 카드

**현재 상태**: 프로그레스 바 + 수치로 간결하게 표시되어 있다. Target Distribution은 스택 바로 표현.

**잘된 점**: 정보가 간결하고 레이아웃이 깔끔하다. "485/830 (58%)" 형태의 표기도 좋다.

**개선 방안**:
- Striking 카드의 Target Distribution 스택 바가 **빨강(Head 71%) + 파랑(Body 24%) + 초록(Leg 5%)**인데, Head가 압도적이어서 나머지가 거의 안 보인다. **최소 너비**를 설정하거나, 스택 바 대신 **3개의 개별 미니 바**로 나누면 비율이 더 잘 보인다
- Grappling 카드에 Takedowns 프로그레스 바가 있지만, **Control Time과 Avg. Control Time은 숫자만 있다**. 체급 평균 대비 바를 추가하면 맥락이 생긴다. 예: Avg. Control Time 2:15에 대해 "체급 평균 1:02" 같은 기준선
- 두 카드 모두 "(45 fights)"라고 표시되어 있는데, 이것이 **UFC 전적인지 전체 MMA 전적인지** 명시하면 좋다

### 11-5. Fight History 테이블

**현재 문제**:
- 상대 선수 이름이 전부 소문자: "jack della maddalena", "renato moicano", "dustin poirier"
- 날짜 형식이 "2025-11-15" (ISO 형식)인데, 사용자 친화적이지 않다
- 별(☆) 아이콘이 있는데 이것이 무엇을 의미하는지 설명이 없다 (즐겨찾기? 타이틀전?)
- WIN 뱃지가 모두 같은 초록색인데, **승리 방법에 따라 색상을 다르게** 하면 시각적으로 풍부해진다

**개선 방안**:
- 상대 이름 Title Case 변환: "Jack Della Maddalena", "Renato Moicano"
- 날짜 형식을 "Nov 15, 2025" 또는 "2025.11.15"로 변경한다. 한국어 사용자 기준이면 "2025.11.15"도 괜찮다
- ☆ 아이콘의 의미를 툴팁으로 설명하거나, 범례를 테이블 상단에 추가한다
- 결과 뱃지에 변형을 준다: WIN(KO/TKO) = 빨간 테두리, WIN(SUB) = 보라 테두리, WIN(DEC) = 시안 테두리. 또는 METHOD 컬럼 자체에 색상을 적용한다
- **상대 선수 이름을 클릭하면 해당 선수의 상세 페이지로 이동**할 수 있어야 한다. 현재 클릭 가능해 보이지 않는다
- 이벤트 이름도 클릭 시 이벤트 상세 페이지로 이동할 수 있으면 탐색성이 올라간다
- "(18)"이라고 되어있는데 화면에는 4경기만 보인다. **스크롤/페이지네이션/더보기**가 있는지 확인 필요. 있다면 괜찮지만, 없다면 추가해야 한다

### 11-6. 페이지 전체 구조에 대한 제안

**빠져있는 정보들**:
- **최근 폼(Recent Form)**: 최근 5경기의 승패를 "W W W W W" 같은 점이나 아이콘으로 표시하는 미니 위젯. 많은 격투기 통계 사이트에서 사용하는 패턴이다
- **체급 내 순위 맥락**: "#1 P4P"와 "Welterweight Champion"이 뱃지로만 있는데, 해당 체급의 **간단한 랭킹 리스트**(상위 5명)를 사이드에 보여주면 맥락이 더 풍부해진다
- **경기 퍼포먼스 추이**: 경기별로 유효타격 수, 테이크다운 성공률 등이 어떻게 변해왔는지 **라인 차트**로 보여주면, 선수의 성장/쇠퇴를 한눈에 볼 수 있다. 이건 장기 개선 항목이다
- **상대 전적 요약**: Fight History 테이블 위에 "vs Top 10: 8-1", "vs Southpaw: 5-0" 같은 **상대 유형별 전적 요약**이 있으면 분석적 가치가 크다

**레이아웃 제안**:
- 현재: 프로필 헤더 → Record + Finish Breakdown (2col) → Striking + Grappling (2col) → Fight History (full width)
- 제안: 프로필 헤더 옆에 **Recent Form 미니 위젯**을 추가하고, Fight History 테이블 위에 **라운드별 퍼포먼스 요약**(평균 유효타격/라운드, 평균 TD/경기 등)을 넣으면 정보 밀도가 올라간다
